Vue.js Parse.js 함께 쓰고 싶은 마음

  • March 9, 2017
  • by Sangyong Gwak

하아... 컴맹이라 별짓하고 있습니다.

Parse Server로부터 데이터를 여러개 가져옵니다.

var Post = Parse.Object.extend("Post");
//var blogs = new Post();
var query = new Parse.Query(Post);
query.limit(10);
query.descending("createdAt");
query.find({
success: querySuccess,
error: queryFailure
});

function querySuccess(posts){
$.each(posts, function(index, value){
console.log(value.createdAt);
console.log(value.get("title"));
});
}

이렇게 접근이 가능한데, Parse Object는 createdAt은 바로 접근이 되는데, 다른 컬럼은 get()을 써야합니다.
그래서, Vue.js에 쓸 때,

var example2 = new Vue({
el: '#example-2',
data: {
items: posts
}
})
라고 하고,

<template v-for="item in items">
<div class="blog-post">
<h2 class="blog-post-title">{{ item.title }}</h2>
<p class="blog-post-meta">{{ item.createdAt }} by <a href="#">imyaman</a></p>
<span>{{ item.content }} {{ item.objectId }}</span>
</div><!-- /.blog-post -->
</template>

이렇게 하면, item.createdAt 만 정상적으로 표시되고, item.title은 정상적으로 표시되지 않습니다.
Vue.js에 쓰기 위해 object array를 다시 만들어보았습니다.

$.each(posts, function(index, value){
str = str + '{"title" : "' + value.get('title') + '"}';
if(index+1 != posts.length){ str=str+','; }
});
str = '[' + str + ']';

postObjects=JSON.parse(str);

var example2 = new Vue({
el: '#example-2',
data: {
items: postObjects
}
})

이렇게 하면 템플릿에서 item.title을 쓸 수 있긴한데, 과연 이게 바른 방법인가... 고민되네요.
http://blog.sys5.co/renew/index.html 파일에서 실험해보고 있습니다.

그 글

안드로이드가 이렇게 흥하게 된 데에는 여러가지 이유가 있겠지만, 앱 스토어 측면에서 이야기 해보려합니다.

피처폰이 한창이던 시절이 있었습니다. 피처폰이 가고 스마트폰이 왔죠. 스마트폰과 함께 iOS와 Android도 왔습니다. 안드로이드는 단지 온게 아니라 엄청난 영향을 미치고 있습니다. 그리고 안드로이드가 이렇게 흥하게 된 것은 통신사의 역할이 큽니다.

전화기(모바일 디바이스)는 통신사의 영역입니다, 전화 서비스는 물론이고요. 이 두 가지는 분리하여 생각하기 어려웠습니다. 요즘에 와서야 전화기를 전자회사에서 살 수 있다는 생각을 하고 있는거지, 전화기는 기본적으로 통신사가 파는 것이었습니다.

통신사는 애초부터 앱 스토어 판의 큰 부분을 차지 하고 있었습니다.
통신사는 피처폰 이전엔 음성 통화, 문자 메시지, 그 외 부가서비스로 돈을 벌었습니다. 피처폰부터 콘텐츠와 데이터 통화로도 돈을 벌게 됩니다. 글, 화보, 벨소리, 음악, 동영상 같은게 콘텐츠죠. 매직엔, 준, 이지아이 같은 버튼을 누르면 피처폰에서 이런 콘텐츠를 이용할 수 있었습니다. 그리고 통신사 입장에서 앱도 콘텐츠로 분류하곤 합니다. 피처폰을 통해 사용자에게 제공되고 각각의 콘텐츠에 가격을 부여하고 이를 전송하는 데이터 통신에 가격을 부여하는 측면에서는 다 비슷한 녀석들입니다. 이것들이 형태는 좀 다를 수 있지만 결국은 앱 스토어입니다.
애플이나 구글처럼 누구나 상품(앱)을 팔 수 있는 오픈마켓이 아니고, 업체(CP, contents provider)로부터 상품을 사서 통신사가 파는 형태였고요.

닌텐도 스위치 사고 싶습니다

  • March 3, 2017
  • by Sangyong Gwak

닌텐도 스위치 사고 싶습니다. 엄청 좋아보이네요. 젤다 게임 동영상 봤는데, 어마어마하게 좋아보입니다.

닌텐도 스위치 구매를 조건으로 대출을 해주면 좋겠습니다. 또는 중고 자동차 매매를 연결해 주어서 닌텐도 스위치 구매 자금을 준비할 수 있도록 도와주는 것도 좋고요. 성형 수술보다 더 좋은 것이 닌텐도 스위치일 것 같아요. 이거 사면 무료 배달 해주어도 좋고, 이왕이면 꽃배달이랑 함께 오면 좋겠어요. 그리고 자동차 보험 가입하듯 닌텐도 스위치 보험도 있으면 좋겠습니다.
